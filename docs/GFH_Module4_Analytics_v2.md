## МОДУЛЬ 4 — Дашборды, иерархическая аналитика и отчёты

**Предусловие:** Модули 1–3 реализованы
**Стек:** Spring Boot + Apache POI (Excel) + iText (PDF) + Spring Cache (Caffeine) + React.js + Recharts

---

### Задача
Реализуй личный дашборд, иерархическую аналитику с тремя контролами, оперативный дашборд руководителя, отдельную аналитику антибонусов и экспорт отчётов.

---

### Требования

#### Цветовая индикация рейтинга (единый стандарт)

| Зона | Условие | Цвет |
|---|---|---|
| Красная | `final_rating < rating_color_red_threshold` | 🔴 |
| Жёлтая | `rating_color_red_threshold ≤ final_rating < rating_color_yellow_threshold` | 🟡 |
| Зелёная | `final_rating ≥ rating_color_yellow_threshold` | 🟢 |

Пороги настраиваются в `system_settings` (`rating_color_red_threshold`, `rating_color_yellow_threshold`).

---

#### Личный дашборд (`/dashboard`)
- Текущий рейтинг с цветовой индикацией (3 зоны)
- Разбивка: отдельная секция «Положительные критерии» (bar chart / таблица) и «Антибонусы» (с красной индикацией)
- График динамики рейтинга, фильтр по типу периода
- Статус активных циклов (все типы периодов — отображаются одновременно)
- Активные апелляции
- Последние уведомления (5 штук)

---

#### Оперативный дашборд руководителя (`/manager-dashboard`)
- Прогресс оценок в текущем цикле «X из N» + прогресс-бар, отдельно по типам периодов
- Сводная таблица подчинённых: рейтинг (с цветовой индикацией), статус оценки, динамика к прошлому периоду
- Топ-3 и антитоп-3 по рейтингу
- Активные апелляции на рассмотрении
- Незакрытые задачи (кого ещё нужно оценить)

---

#### Иерархическая аналитика (`/analytics/hierarchical`) — главная фича

**Три независимых контрола:**

| Контрол | Варианты |
|---|---|
| Режим отображения | Таблица / Карточки / Развёрнутое дерево / График сравнения |
| Разрез (уровень агрегации) | Организация / Блоки / Департаменты / Отделы / Сотрудники |
| Фильтр подразделений | Multi-select с учётом прав роли (по умолчанию все доступные) |

- Для каждой группы отображаются две цифры: «Средний рейтинг группы» и «Рейтинг руководителя группы» (с пометкой, что это разные показатели)
- Чекбокс «Исключить уволенных» (по умолчанию включены)
- Доступ к разрезам ограничен правами роли согласно разделу 8.2 `GFH_Part1_General.md`
- **ADMIN** видит оценки всех сотрудников без ограничений

**Drill-down:**
- Клик на строку / карточку / узел дерева → модальное окно с деталями
- В модальном: подчинённые с рейтингами, разбивка по критериям, список применённых антибонусов
- Модальное окно не меняет состояние основной страницы (URL не меняется)

**Сравнительный режим (переключатель сверху):**
- Сравнение подразделений (выбор 2–4 для параллельного показа)
- Сравнение с предыдущим периодом того же типа
- Benchmark (текущее подразделение vs средний уровень вышестоящей группы)
- Каждый режим доступен в любом из четырёх режимов отображения

**Сохранение настроек в `localStorage`:**
```
analytics_mode        — режим отображения
analytics_grouping    — разрез
analytics_units       — выбранные подразделения
analytics_compare     — режим сравнения
analytics_include_inactive — чекбокс уволенных
```
При следующем входе страница открывается в том же виде.

---

#### Аналитика антибонусов (`/analytics/anti-bonuses`)
- Сотрудники группы, отсортированные по суммарному `anti_bonus_penalty` за период
- Топ-10 получивших больше всего штрафов
- Распределение антибонусов по критериям (какие нарушения чаще всего)
- Динамика за последние 12 периодов
- Отдельная статистика автоматических антибонусов
- Фильтры: период, тип антибонуса (ручной / автоматический), подразделение

---

#### Производительность
- Вся агрегация на уровне БД (JPQL / native SQL), никаких выгрузок всех записей на фронт
- Кеширование аналитики: Spring Cache + Caffeine (TTL 5 минут, инвалидация при закрытии периода)
- Индексы: `evaluations(period_id, evaluatee_id)`, `evaluations(evaluator_id, period_id)`, `evaluation_scores(evaluation_id, criteria_id)`, `evaluation_scores(criteria_id)`

---

#### Адаптивность (Responsive React)
- Все компоненты работают на мобильном экране (responsive Tailwind, без React Native)
- На мобильном три контрола сворачиваются в выпадающую панель настроек

---

#### Экспорт отчётов
- Форматы: **Excel** (Apache POI) + **PDF** (iText)
- Все отчёты содержат: логотип ГФХ (placeholder), дату формирования, имя сформировавшего, параметры фильтра
- Кнопки экспорта на каждой аналитической странице
- Факт экспорта фиксируется в `audit_log`: кто, когда, какой отчёт, параметры

---

### API endpoints

| Метод | Endpoint | Описание |
|---|---|---|
| GET | `/api/v1/analytics/my-dashboard?periodType=` | Личный дашборд |
| GET | `/api/v1/analytics/manager-dashboard?periodType=` | Оперативный дашборд руководителя |
| GET | `/api/v1/analytics/hierarchical` | Иерархическая аналитика (params: mode, grouping, unitIds, periodType, periodId, compareMode, compareWith, includeInactive) |
| GET | `/api/v1/analytics/hierarchical/detail?entityType=&entityId=&periodId=` | Drill-down модальное окно |
| GET | `/api/v1/analytics/anti-bonuses?periodId=&unit=&type=` | Аналитика антибонусов |
| GET | `/api/v1/analytics/comparison?units=&periodType=` | Сравнение подразделений |
| GET | `/api/v1/analytics/dynamics?entityType=&entityId=&periodType=&from=&to=` | Динамика |
| GET | `/api/v1/reports/export?reportType=&format=excel|pdf&params=` | Экспорт отчёта |

### Ожидаемый результат
- Spring Boot: `AnalyticsService`, `HierarchicalAnalyticsService`, `AntiBonusAnalyticsService`, `ReportService`
- Агрегационные SQL-запросы на уровне БД (без выгрузки всего на фронт)
- Spring Cache + Caffeine для аналитических запросов (TTL 5 мин)
- React.js (Vite + Redux Toolkit + React Router v6 + Tailwind + shadcn/ui + Recharts):
  - Личный дашборд: цветовая индикация рейтинга (3 зоны), разделение положительных и антибонусов
  - Оперативный дашборд руководителя: прогресс-бар, топ/антитоп, задачи
  - Страница иерархической аналитики: три контрола, четыре режима отображения, сравнительный режим, drill-down
  - Модальное окно drill-down (без изменения URL)
  - Страница аналитики антибонусов: топы, динамика, фильтры
  - Сохранение настроек аналитики в `localStorage`
  - Адаптивная вёрстка (mobile-first Tailwind)
  - Кнопки экспорта Excel/PDF на каждой странице
- Интеграционные тесты (Testcontainers): агрегационные запросы, экспорт
